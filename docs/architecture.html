关键设计点（答辩话术）

  • Supervisor pattern：所有 agent 输出汇到 PM，PM 做内联 QA + 路由 → 集中可观测性 + 单点调试
  • 跨家族 self-preference bias 缓解：PM=DeepSeek 异于 Report=GPT-5；终审 Doubao 异于两者，三家族对抗
  • Collector / Insight 并行 fan-out：互不依赖，LangGraph 原生并发，省时间
  • 动态任务管理：TaskRegistry 跟踪每个 task 的 status / retry_count / output
  • 可信度规则客观：retry > 2 即 forced，写入 unreviewed，报告自动列出，不靠 LLM 主观打分
  • PM 不重复 Collector 的 web tool：单一职责，事实核查归 Collector，PM 只审 QA

5 角色映射

  • 📋 PM Agent → DeepSeek，supervisor + inline QA + TaskRegistry
  • 📦 Collector → DeepSeek（文本）+ GPT-5（多模态分支）
  • 💬 Insight → DeepSeek，挂 questionnaire skill
  • 📊 Analyst → DeepSeek，NLP + 打分 + SWOT
  • 📝 Report → GPT-5 多模态，最终 MD + PDF 渲染
