**背景**：Reporter 跑完全部分析工具后，Doubao 输出"。"停了，没有写报告正文也没调 `render_pdf`。根本原因：Doubao 上下文过长，输出被截断。

- `qa_report.py` 加 `_fallback_generate_report()`：ReAct 结束后若 `report_md` 仍为空，提取已完成的排名 / SWOT / 图表工具产出，单次 LLM 调用生成兜底报告
- `Reporter` 的 `recursion_limit` 从 40 调至 80

**涉及文件**：`src/cca/prompts/report_agent.md`、`src/cca/agents/qa_report.py`

## 五、修复 JSON 截断无限重试

**背景**：Doubao 生成超长 profile JSON 时被 token 上限截断，`finalize_profile` 持续报错，模型无限循环 8–10 次。

**完成**：

- `_validation.py` 新增三阶段 JSON 修复流程：
  1. 标准 `json.loads`
  2. Extra data 尾部杂质 → `raw_decode` 取首个完整对象
  3. 中途截断 → `_try_recover_truncated()`：检测是否在字符串内部结束（`_scan_structure`），从末尾回退最多 1200 字符，自动补 `"` + `}` / `]` 恢复截断 JSON
- `collector_tools.py` `finalize_profile` 用 `pre_clean` 自动注入外层 `product_name`，消除"Field required"死循环

**涉及文件**：`src/cca/tools/_validation.py`、`src/cca/tools/collector_tools.py`
